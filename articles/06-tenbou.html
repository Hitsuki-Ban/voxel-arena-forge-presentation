LLMに「作らせて、検証する」 第6回(最終回) / シリーズ目次

で、ゲーム開発のどこから使うか

ここまで、型(第2回)、やり取り(第3回)、検査と計測(第4回)、開発体制(第5回)と書いてきました。 最終回は、この実験を自分たちの現場に持ち帰るとしたら何から始めるか — と、 あわせてこの実験がまだできていないことを、正直に書きます。

向いている領域の条件

「作らせて検証する」方式が向くのは、出力がデータやスクリプトの形に落ちるコンテンツです。条件は3つに要約できます。

  1. 機械検証できること。 スキーマ・型検査・自動テスト・シミュレーションなど、「通った/弾かれた」を機械が判定できる。 これがないと、結局人間が全件レビューすることになり、量産の意味がなくなります。
  2. 失敗が安くて速いこと。 1回の生成と検証が数十秒〜数分で回り、失敗してもゲームを壊さない。 この実験では受け入れ1体あたり約1.4万トークン、生成から検証完了まで10〜20秒でした。
  3. 人の仕事を「仕様の対話」と「レビュー」に寄せられること。 白紙から書く工程を AI に移し、人は方向を選び、結果を見極める。 候補カード(第3回)のような意思決定支援があると、この移行が滑らかになります。 生成が理想形に届かない場合も、様式が揃っていれば叩き台として工程は前に進みます(第2回)。

具体的には、スキル・行動スクリプト、装備・敵などのマスタデータ、配置・エンカウント、バランス初期値あたりが最初の候補です。

投資の順序 — モデル選定は最後

この実験の一番の学びを一行にすると「型 → 検査 → 軽量モデルの順に投資する」です。

順序作るもの現場での原料
1. 型(原型箱)コマンド/API のメニュー、データスキーマ、素材台帳、通った実例と失敗の記録プロジェクトの API 定義、マスタデータ定義、アセット管理台帳、レビュー指摘の記録 — ほぼ既存資産
2. 検査スキーマ検証、静的検査、自動テスト、できれば決定論の実行環境既存の CI・テスト・シミュレータ/リプレイ基盤
3. モデル役割ごとに選ぶ。対話=軽量で成立、実装=軽量でも中量でも検査が担保する限り、単価の安い方へ差し替えた分がまるごと削減(第4回)

逆順 — 先に高いモデルを契約して、型と検査を後回しにする — は、この実験では機能しませんでした。 重量級モデルに替えても数字が動かなかった時期(第2回)がその証拠です。

情報の扱い

外部 API に送っているのは、仕様の相談内容と型の定義(プロンプト)だけです。 ソースコード全体やマスタデータ一式は送りません。 さらに接続先は OpenAI 互換 API なので、ベース URL の設定ひとつで差し替えられます。 実際に 1.5B の小型オープンモデル(Qwen2.5-Coder + llama.cpp)をローカルで立て、 仕様相談から生成・検証までの全工程が通ることを実機で確認しました。 機密度の高いタイトルでは、オンプレのモデルで同じ仕組みを回す選択肢が最初からあります。

この実験がまだできていないこと

限界も正直に書いておきます。

  • 戦闘は少数戦(1v1〜7体)まで。大規模戦・地形・視線などは扱っていません。
  • コマンド語彙は16種で、回復のような概念はまだ表現できません。語彙を増やすたびに型と失敗パターン集の再調整が要ります。
  • 受け入れ検査は「壊れていない・ルールに沿っている」を保証しますが、「面白い」は保証しません。面白さの評価は依然として人間の仕事です。
  • 数字は特定のモデル(deepseek-v4 世代)と日本語 UI での実測です。別モデル・別言語では再計測が必要です。

特に3点目は強調しておきたい点です。この仕組みが変えるのは「動くものを量産する」工程で、 「良いものを選ぶ」工程は人間の判断に残ります。だからこそ、候補カードやバランスレポートのような 選ぶ側の道具に投資する価値があります。

次にやるなら

  • プロジェクト資産の型化パイロット — いま動いているプロジェクトのマスタ定義+アセット台帳を1領域だけ「LLM が読める契約」に固め直し、10件生成ゲートを作って通過率を測る。この実験の再現で、最小の一歩です。
  • 体験会形式 — 参加者が自分でお題を入れてユニットを作る会。工房はそのまま使えます。
  • チーム戦・より大きな戦闘 — 決定論エンジンの範囲を広げ、量産したユニットの「編成」まで実験対象にする。

このシリーズが、「LLM をどう使うか」の議論を「何を型にして、何で検査するか」の議論に 変換する材料になれば、書いた甲斐があります。

手元で試す: リポジトリで pnpm install → pnpm demo:local:mock(API 不要)。実 API を使う場合は pnpm demo:local。
